HyperCheckout · Introduction

Ship payments with one integration

HyperCheckout is a fully managed, native, cross-platform checkout that blends with your app or website. Your backend creates a session and confirms the result; the SDK renders the payment screen. Integrate once, then enable methods, routing, and retries through configuration rather than new code.

● Production ready Hosted UI PCI-DSS handled Android · iOS · Web · React Native

What is HyperCheckout

HyperCheckout is a payments orchestrator, not just a checkout UI. You integrate once and get a conversion-optimised payment experience you don't have to build, plus the ability to route volume across multiple gateways. That means fewer app releases for payments, quicker reconciliation, and easier refunds and customer queries.

You only ever touch two surfaces:

BACKEND

Your server

Create a session, receive webhooks, and read order status. Identical across every platform.

FRONTEND

The SDK

Open the checkout screen with the session token and read the result. Swap only this per platform.

How a payment flows

Map your own code onto these steps. Blue runs on your backend; orange runs in the SDK. The backend half is identical across Android, iOS, Web, and React Native.

1
BECustomer hits checkout. Your server calls Session API POST /session with order id, amount, and currency.
2
BEJuspay returns a client_auth_token. Pass it to your client — never your API key.
3

4
FECustomer pays. HyperCheckout handles method selection, 3DS, retries, and routing.
5

!
The SDK callback can be missed if the app is killed mid-payment. The webhook + Order Status API are the source of truth. Always reconcile server-side before fulfilling.

Good to know

Three HyperCheckout behaviours that affect how you build and what you should expect at runtime.

Dynamic routing runtime

When you connect more than one payment processor, Juspay can route each transaction to the best-performing gateway automatically. It's why a payment may land on a different gateway than you'd expect. The mechanism:

  • A score from 0 to 1 is maintained per gateway-and-payment-method combination, in a 30-minute window.
  • The score drops when a transaction starts and rises when it succeeds.
  • When a score falls below your configured threshold, routing moves to another gateway that supports the method with a better score.

Requires association with more than one payment processor. Works alongside health-based and static routing rules.

Over-the-air updates (OTA) releases

Configuration changes — styling, reordering payment methods — and many new payment features ship as over-the-air updates, without an app release. Plan your payment changes around this: you don't need an app-store submission to reorder methods or restyle the sheet.

One-click payments Android only

VISA Safe Click is a network-authentication one-click flow for repeat users on low-value transactions. It completes faster than a full 3DS flow. Note the constraint: it's available on Android only and applies below a value threshold, so don't assume it on iOS or Web.

Before you start

  • A Merchant ID and API key from the Juspay Dashboard (start in sandbox).
  • A webhook URL configured to receive authoritative outcomes.
  • Sandbox and production use different base URLs and keys — sandbox.juspay.in vs api.juspay.in. Swap both to go live.
i
The response field that matters is client_auth_token — pass it to the client. It expires in 15 minutes and is scoped to one order.
